## High-level description
This file is a TypeScript configuration file (tsconfig.json) for the api-sdk package. It defines compiler options and project settings to ensure strict type-checking, enable modern JavaScript features, and configure output for the TypeScript compiler.

## Code Structure
The configuration is structured as a JSON object with three main sections: "extends", "compilerOptions", and file inclusion/exclusion settings.

## Symbols

### Root Configuration Object
#### Description
The root object contains the entire TypeScript configuration for the api-sdk package.

#### Internal Logic
1. Extends the recommended TypeScript configuration.
2. Specifies compiler options for strict type-checking and modern JavaScript features.
3. Defines input and output directories for the compilation process.

### "extends"
#### Description
Specifies the base configuration to extend from.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| @tsconfig/recommended/tsconfig.json | string | Path to the recommended TypeScript configuration |

### "compilerOptions"
#### Description
Defines the compiler options for the TypeScript compiler.

#### Internal Logic
1. Sets the target ECMAScript version to ES2022.
2. Enables source map generation.
3. Configures declaration file generation.
4. Enables decorator metadata and experimental decorators.
5. Incorporates strict type-checking options from @tsconfig/strictest.
6. Specifies the output directory for compiled files.

### "include"
#### Description
Specifies which files to include in the compilation.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| ./src | string | Path to the source directory |

### "exclude"
#### Description
Specifies which files or directories to exclude from compilation.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| dist | string | Output directory |
| node_modules | string | Dependencies directory |
| **/__tests__ | string | Test files (using a glob pattern) |

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| @tsconfig/recommended | Provides a base recommended TypeScript configuration |

## Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| target | string | "ES2022" | Specifies the ECMAScript target version |
| sourceMap | boolean | true | Enables generation of source map files |
| declaration | boolean | true | Generates corresponding .d.ts files |
| declarationMap | boolean | true | Generates a sourcemap for .d.ts files |
| emitDecoratorMetadata | boolean | true | Enables experimental support for emitting decorator metadata |
| experimentalDecorators | boolean | true | Enables experimental support for decorators |
| allowUnusedLabels | boolean | false | Disallows unused labels |
| allowUnreachableCode | boolean | false | Disallows unreachable code |
| noFallthroughCasesInSwitch | boolean | true | Reports errors for fallthrough cases in switch statements |
| noImplicitOverride | boolean | true | Ensures overriding members are marked with an override modifier |
| noUnusedLocals | boolean | true | Reports errors on unused local variables |
| checkJs | boolean | true | Enables type checking on JavaScript files |
| exactOptionalPropertyTypes | boolean | true | Enables strict checking of optional property assignments |
| noImplicitReturns | boolean | true | Ensures all code paths in a function return a value |
| outDir | string | "./dist" | Specifies the output directory for compiled files |

This configuration sets up a strict TypeScript environment for the api-sdk package, ensuring type safety, enabling modern JavaScript features, and configuring the compilation process to generate declaration files and source maps for better developer experience.
