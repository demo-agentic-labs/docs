## High-level description
This Handlebars template file is designed to map medication request status codes from a CDA (Clinical Document Architecture) format to FHIR (Fast Healthcare Interoperability Resources) MedicationRequest status codes. It provides a standardized way to convert various status codes to their FHIR equivalents.

## Symbols

### Main Template
#### Description
The main template takes a `code` input and converts it to a corresponding FHIR MedicationRequest status.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| code | string | The input status code from the CDA format |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| status | string | The corresponding FHIR MedicationRequest status |

#### Internal Logic
1. The input `code` is converted to lowercase using the `toLower` helper.
2. The normalized code is then compared against a series of predefined values using `if-else` statements.
3. Each match returns the corresponding FHIR MedicationRequest status.
4. If no match is found, it returns "unknown".

## Dependencies
This template relies on the Handlebars templating engine and its built-in helpers.

## Configuration
The template uses the following mapping for status codes:

| CDA Status | FHIR MedicationRequest Status |
|:-----------|:------------------------------|
| normal | active |
| aborted | stopped |
| active | active |
| cancelled | cancelled |
| completed | completed |
| held | on-hold |
| new | active |
| suspended | on-hold |
| nullified | entered-in-error |
| obsolete | stopped |
| (default) | unknown |

## References
The template includes references to the following specifications:
1. HL7 CCDA on FHIR Implementation Guide: ConceptMap-CF-MedicationStatus
2. HL7 Terminology: v3-ActStatus ValueSet
3. FHIR R4 Specification: MedicationRequest Status ValueSet

These references provide the basis for the status code mappings implemented in the template.
