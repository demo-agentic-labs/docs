## High-level description
This Handlebars template file defines a mapping for request status codes. It translates various input status codes to their corresponding FHIR RequestStatus values, providing a standardized output for different status representations.

## Symbols

### RequestStatus Mapping
#### Description
This template uses a series of conditional statements to map input status codes to FHIR RequestStatus values. It covers various possible status inputs and provides a default "unknown" status for unrecognized inputs.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| code | string | The input status code to be mapped |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| status | string | The corresponding FHIR RequestStatus value |

#### Internal Logic
The template uses a series of `{{#if}}` and `{{else if}}` statements to check the input `code` against known values. Each condition maps to a specific FHIR RequestStatus:

1. "draft" maps to "draft"
2. "active" maps to "active"
3. "suspended" or "on-hold" map to "on-hold"
4. "failed" or "revoked" map to "revoked"
5. "completed" maps to "completed"
6. "entered-in-error" or "error" map to "entered-in-error"
7. "unknown" maps to "unknown"
8. Any other input defaults to "unknown"

This mapping ensures that various input statuses are standardized to the FHIR RequestStatus vocabulary.

## Dependencies
This template relies on the Handlebars templating engine for its functionality.
| Dependency | Purpose |
|:-----------|:--------|
| Handlebars | Templating engine used for conditional logic and output generation |

## Error Handling
The template includes a default case that maps any unrecognized input to "unknown", effectively handling potential errors or unexpected inputs.
