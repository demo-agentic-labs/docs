## High-level description
This Handlebars template file defines a mapping for identifier types based on OID (Object Identifier) codes. It's designed to categorize different health information system vendors (eClincicalWorks, Cerner, and Epic) based on the prefix of the provided code.

## Symbols

### Main Template
#### Description
The main template checks if a `code` is provided and then determines the corresponding health information system vendor based on the code's prefix.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| code | string | An OID code representing a health information system vendor |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| JSON object | object | A JSON object with an "oid" property containing the vendor name |

#### Internal Logic
1. Check if `code` exists.
2. If `code` exists, create a JSON object with an "oid" property.
3. Use conditional statements to determine the vendor based on the code prefix:
   - If the code starts with '2.16.840.1.113883.4', set "oid" to "eClincicalWorks"
   - If the code starts with '2.16.840.1.113883.3', set "oid" to "Cerner"
   - If the code starts with '1.2.840.114350', set "oid" to "Epic"

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| Handlebars | Template engine used for conditional logic and output generation |

## Configuration
The template uses Handlebars helpers:
| Option | Type | Description |
|:-------|:-----|:------------|
| startsWith | helper | Custom Handlebars helper to check if a string starts with a given prefix |
| if | helper | Built-in Handlebars helper for conditional rendering |

## Error Handling
The template does not implement explicit error handling. If the `code` doesn't match any of the defined prefixes, the "oid" property will be empty.
