## High-level description
This Handlebars template defines a partial FHIR ActivityDefinition resource with a focus on the `specimenRequirement` field. It's designed to be used in the context of converting Clinical Document Architecture (CDA) data to FHIR format.

## Symbols

### ActivityDefinition Template
#### Description
This template creates a partial FHIR ActivityDefinition resource structure with a `specimenRequirement` array containing a single reference.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| ID | String | The identifier for the ActivityDefinition resource |
| REF | String | The reference value for the specimen requirement |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| JSON object | Object | A partial FHIR ActivityDefinition resource with `specimenRequirement` |

#### Internal Logic
1. Creates an outer `resource` object.
2. Sets the `resourceType` to "ActivityDefinition".
3. Assigns the provided `ID` to the `id` field.
4. Creates a `specimenRequirement` array with a single object.
5. Sets the `reference` field of the specimen requirement to the provided `REF` value.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| Handlebars | Template engine used for generating the JSON structure |
