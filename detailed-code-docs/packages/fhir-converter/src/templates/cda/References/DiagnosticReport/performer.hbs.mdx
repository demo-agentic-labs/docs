## High-level description
This Handlebars template file defines the structure for creating a FHIR DiagnosticReport resource with a focus on the performer field. It's designed to convert CDA (Clinical Document Architecture) data to FHIR format, specifically for the DiagnosticReport's performer information.

## Symbols

### Main Template
#### Description
The main template creates a FHIR DiagnosticReport resource structure with emphasis on the performer field. It includes the resource's fullUrl, id, code, status, and performer.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| ID | String | Unique identifier for the DiagnosticReport |
| diagReport | Object | Contains diagnostic report data including code, title, and statusCode |
| REF | String | Reference to the performer of the diagnostic report |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| JSON | Object | A FHIR-compliant DiagnosticReport resource JSON structure |

#### Internal Logic
1. Sets the fullUrl and id using the provided ID.
2. Uses partial templates to populate the code and status fields:
   - `DataType/CodeableConcept.hbs` for the code field
   - `ValueSet/DiagnosticReportStatus.hbs` for the status field
3. Sets the performer as an array with a single object containing a reference.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| DataType/CodeableConcept.hbs | Used to generate the code field |
| ValueSet/DiagnosticReportStatus.hbs | Used to generate the status field |

## Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| canBeUnknown | Boolean | true | Allows the code to be unknown in the CodeableConcept partial |
