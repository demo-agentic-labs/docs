## High-level description
This Handlebars template defines a partial FHIR CapabilityStatement resource structure, specifically focusing on the implementation custodian reference. It's designed to be used in generating FHIR-compliant JSON representations of CapabilityStatement resources.

## Symbols

### CapabilityStatement Template
#### Description
This template creates a partial JSON structure for a FHIR CapabilityStatement resource, specifically defining the implementation custodian reference.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| ID   | String | The identifier for the CapabilityStatement resource |
| REF  | String | The reference to the custodian of the implementation |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| JSON | Object | A partial FHIR CapabilityStatement resource in JSON format |

#### Internal Logic
1. Creates a nested JSON structure representing a FHIR CapabilityStatement resource.
2. Sets the `resourceType` to "CapabilityStatement".
3. Uses the `{{ID}}` input to set the `id` field of the resource.
4. Defines the `implementation` object with a nested `custodian` object.
5. Sets the `reference` field of the `custodian` object using the `{{REF}}` input.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| Handlebars | Template engine used for generating the JSON structure |
