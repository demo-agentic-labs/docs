## High-level description
This Handlebars template defines a partial FHIR PaymentReconciliation resource structure, specifically focusing on the "requestor" field. It's designed to be used in the context of converting Clinical Document Architecture (CDA) data to FHIR format.

## Symbols

### PaymentReconciliation Template
#### Description
This template creates a partial FHIR PaymentReconciliation resource with a focus on the "requestor" field. It sets up the basic structure of the resource and includes placeholders for dynamic data.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| ID | String | The unique identifier for the PaymentReconciliation resource |
| REF | String | The reference to the requestor of the payment reconciliation |

#### Outputs
A JSON object representing a partial FHIR PaymentReconciliation resource.

#### Internal Logic
1. Sets the `resourceType` to "PaymentReconciliation"
2. Assigns the provided `ID` to the resource's `id` field
3. Creates a `requestor` object with a `reference` field, populated by the `REF` input
