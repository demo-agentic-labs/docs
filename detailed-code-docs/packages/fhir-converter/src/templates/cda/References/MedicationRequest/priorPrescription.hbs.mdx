## High-level description
This Handlebars template defines a FHIR MedicationRequest resource with a priorPrescription reference. It's designed to be used in the context of converting CDA (Clinical Document Architecture) data to FHIR (Fast Healthcare Interoperability Resources) format.

## Symbols

### MedicationRequest Template
#### Description
This template creates a FHIR MedicationRequest resource with a priorPrescription reference. It's structured to be part of a larger FHIR bundle or response.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| ID | String | The identifier for the MedicationRequest resource |
| REF | String | The reference to the prior prescription |

#### Outputs
A JSON object representing a FHIR MedicationRequest resource with a priorPrescription reference.

#### Internal Logic
1. Creates a resource object with type "MedicationRequest"
2. Sets the resource ID using the provided `{{ID}}` input
3. Adds a priorPrescription object with a reference to `{{REF}}`

## Dependencies
This template relies on the Handlebars templating engine to process the placeholders ({{ID}} and {{REF}}).

## Configuration
This template doesn't include any specific configuration options. It uses predefined placeholders that should be filled with appropriate values when the template is rendered.

## Error Handling
There's no explicit error handling in this template. It assumes that the ID and REF values will be provided correctly when the template is used.
