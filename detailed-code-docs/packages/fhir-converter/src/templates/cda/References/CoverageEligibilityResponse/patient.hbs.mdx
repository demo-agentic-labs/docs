## High-level description
This Handlebars template defines a partial FHIR CoverageEligibilityResponse resource structure, specifically focusing on the patient reference within the resource. It's designed to be used as part of a larger template for converting CDA (Clinical Document Architecture) data to FHIR format.

## Symbols

### CoverageEligibilityResponse Template
#### Description
This template creates a partial JSON structure for a FHIR CoverageEligibilityResponse resource, with emphasis on the patient reference.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| ID | String | The unique identifier for the CoverageEligibilityResponse resource |
| REF | String | The reference to the patient associated with this CoverageEligibilityResponse |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| JSON structure | Object | A partial FHIR CoverageEligibilityResponse resource in JSON format |

#### Internal Logic
1. Creates a nested JSON structure with a "resource" object.
2. Sets the "resourceType" to "CoverageEligibilityResponse".
3. Assigns the provided ID to the "id" field.
4. Creates a "patient" object with a "reference" field, populated by the REF input.
