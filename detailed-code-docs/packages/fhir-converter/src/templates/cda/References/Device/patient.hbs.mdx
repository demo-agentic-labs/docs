## High-level description
This Handlebars template defines a FHIR Device resource with a patient reference. It's designed to create a JSON structure representing a medical device associated with a specific patient in a FHIR-compliant format.

## Symbols

### Device Resource Template
#### Description
This template creates a JSON object representing a FHIR Device resource. It includes the resource type, an ID, and a patient reference.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| ID | String | The unique identifier for the Device resource |
| REF | String | The reference to the associated patient |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| JSON Object | Object | A FHIR-compliant Device resource JSON structure |

#### Internal Logic
1. Sets the `resourceType` to "Device"
2. Assigns the provided `ID` to the `id` field
3. Creates a `patient` object with a `reference` field containing the provided `REF` value

## Dependencies
This template relies on the Handlebars templating engine to process the placeholders ({{ID}} and {{REF}}).

## Configuration
No specific configuration options are defined in this template.

## Error Handling
This template does not implement any explicit error handling. It assumes that the `ID` and `REF` values will be provided when the template is rendered.

## API/Interface Reference
This template expects two input variables to be provided when rendered:

| Variable | Description |
|:---------|:------------|
| ID | The unique identifier for the Device resource |
| REF | The reference to the associated patient |

The output is a JSON object representing a FHIR Device resource with the provided ID and patient reference.
