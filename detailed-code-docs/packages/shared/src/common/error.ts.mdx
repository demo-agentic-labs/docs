## High-level description
This code defines utility functions for error handling and string conversion in TypeScript. It provides methods to convert errors to strings with different levels of detail and options for customization.

## Symbols

### `ErrorToStringOptions`
#### Description
A type definition for options used in error to string conversion.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| detailed | boolean | Determines whether to use detailed error conversion |

### `errorToString`
#### Description
Converts an error to a string representation, with an option for detailed output.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| err | unknown | The error to be converted |
| options | ErrorToStringOptions | Options for conversion (default: \{ detailed: true \}) |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| result | string | String representation of the error |

#### Internal Logic
- If `options.detailed` is true, calls `detailedErrorToString`
- Otherwise, calls `genericErrorToString`

### `genericErrorToString`
#### Description
Converts an error to a basic string representation.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| err | unknown | The error to be converted |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| result | string | Basic string representation of the error |

#### Internal Logic
- Returns the `message` property of the error if it exists
- Otherwise, converts the error to a string using `String()`

### `detailedErrorToString`
#### Description
Converts an error to a detailed string representation, including additional info and cause if available.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| error | any | The error to be converted |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| result | string | Detailed string representation of the error |

#### Internal Logic
- Extracts the error message
- Includes additional info if available
- Recursively includes cause information if present
