## High-level description
This TypeScript file provides utility functions for boolean value checking. It focuses on determining if a given value is truly "true", not just truthy, with special handling for string and boolean inputs.

## Symbols

### `isTrue`
#### Description
This function determines if a given value is truly "true". It handles different types of inputs, specifically strings and booleans, with a fallback for other types.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| value | unknown | The value to be checked |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| result | boolean | True if the input is considered true, false otherwise |

#### Internal Logic
1. If the input is falsy (null, undefined, 0, "", etc.), it returns false.
2. If the input is a string, it delegates to the `isTrueString` function.
3. If the input is a boolean, it returns the boolean value directly.
4. For any other type, it returns false.

### `isTrueString`
#### Description
This function checks if a given string value represents "true". It normalizes the input string before comparison.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| value | string | The string value to be checked |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| result | boolean | True if the input string represents "true", false otherwise |

#### Internal Logic
1. If the input is falsy (empty string, null, undefined), it returns false.
2. It trims the input string and converts it to lowercase.
3. It compares the normalized string to "true" and returns the result of this comparison.

## References
The `isTrue` function references the `isTrueString` function internally.
