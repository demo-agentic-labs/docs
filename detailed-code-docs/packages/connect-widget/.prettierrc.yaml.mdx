## Symbols

### `.prettierrc.yaml`
#### Description
This YAML file contains configuration options for Prettier, a popular code formatting tool. It sets various formatting preferences for the connect-widget package.

#### Internal Logic
The file defines the following Prettier options:

1. `printWidth: 100`: Sets the line length that the formatter will wrap on to 100 characters.
2. `tabWidth: 2`: Specifies that each indentation level should use 2 spaces.
3. `useTabs: false`: Instructs Prettier to use spaces for indentation instead of tabs.
4. `semi: true`: Ensures that semicolons are added at the end of statements.
5. `singleQuote: false`: Configures Prettier to use double quotes for strings instead of single quotes.
6. `trailingComma: "es5"`: Adds trailing commas where valid in ES5 (objects, arrays, etc.).
7. `bracketSpacing: true`: Adds spaces between brackets in object literals.
8. `arrowParens: "avoid"`: Omits parentheses around a sole arrow function parameter when possible.

## Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| printWidth | number | 100 | The line length where Prettier will try to wrap |
| tabWidth | number | 2 | Number of spaces per indentation level |
| useTabs | boolean | false | Whether to use tabs instead of spaces |
| semi | boolean | true | Whether to add semicolons at the end of statements |
| singleQuote | boolean | false | Whether to use single quotes instead of double quotes |
| trailingComma | string | "es5" | Where to print trailing commas |
| bracketSpacing | boolean | true | Whether to print spaces between brackets in object literals |
| arrowParens | string | "avoid" | When to add parens around arrow function parameters |
