## High-level description
The `WidgetContainer` component is a reusable container for the Metriport connect widget. It provides a consistent layout and styling for the widget's content, including a header with the Metriport logo, a scrollable content area, and dynamic theming based on custom colors and color mode.

## Code Structure
The main `WidgetContainer` component uses various Chakra UI components and hooks to create the layout. It also utilizes React hooks for state management and side effects. The component is wrapped in a `ChakraProvider` with a custom theme.

## Symbols

### WidgetContainer
#### Description
A functional component that serves as a container for the Metriport connect widget. It handles custom theming, responsive layout, and scroll behavior.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| children | JSX.Element | The content to be rendered inside the container |

#### Internal Logic
1. Retrieves custom color from URL parameters and local storage
2. Creates a custom theme using Chakra UI's `extendTheme`
3. Manages the visibility of a scroll indicator icon
4. Handles scroll behavior and resets scroll position when children change
5. Renders a responsive layout with a header, scrollable content area, and optional scroll indicator

## Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| CUSTOM_COLOR_PARAM | string | "customColor" | URL parameter for custom color |
| PRIMARY_COLOR | string | "#748df0" | Default primary color |
| HOVER_COLOR | string | "#879ced" | Default hover color |
| BREAKPOINTS | object | \{ xs: "300px", ... \} | Responsive breakpoints |

## Side Effects
- Modifies the scroll position of the content area
- Updates the visibility of the scroll indicator based on content and container size

## Performance Considerations
The component uses `useEffect` hooks to manage scroll behavior and icon visibility, which may impact performance if the content changes frequently or is very large.
