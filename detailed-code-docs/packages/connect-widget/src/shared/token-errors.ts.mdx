## High-level description
This code defines custom error classes related to token handling in a Connect Widget. It provides specific error types for scenarios such as missing tokens, demo tokens, and invalid tokens, each with customizable messages, titles, and links to relevant documentation.

## Symbols

### `NoTokenError`
#### Description
A custom error class for scenarios where the Connect Token is missing.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| message | string (optional) | Custom error message |
| link | string (optional) | Custom documentation link |
| icon | string (optional) | Custom icon for the error |

#### Internal Logic
- Sets a default error message if none is provided
- Sets a default documentation link if none is provided
- Sets a default title "Missing Connect Token"

## References
These error classes are used in the related file `packages/connect-widget/src/shared/api.ts`:

- `NoTokenError` is thrown when the API token is missing from the query parameters.
- `DemoTokenError` is thrown when a demo token is detected.
- `InvalidTokenError` is thrown when the provided token is invalid.

## Dependencies
This code doesn't have any external dependencies. It extends the built-in `Error` class.

## Error Handling
This file defines custom error classes that are used for specific token-related error scenarios in the Connect Widget. These classes provide structured error information, including titles, messages, and links to relevant documentation, which can be used to display user-friendly error messages in the UI.
