## High-level description
This code defines default values and types for sandbox seed data used in a healthcare-related application. It exports a bucket name and a `DataEntry` type, which includes information about S3 storage and FHIR DocumentReference resources.

## Symbols

### `bucket`
#### Description
A constant that holds the name of the S3 bucket used for storing sandbox seed data. It uses the `Config.getSandboxSeedBucketName()` method to retrieve the bucket name, with a fallback value of "missing-value" if the method returns null or undefined.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| bucket | string | The name of the S3 bucket for sandbox seed data |

### `DataEntry`
#### Description
A TypeScript type definition that represents a single entry of sandbox seed data. It combines information about S3 storage and a FHIR DocumentReference resource.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| s3Info | object | Information about the S3 storage location |
| docRef | DocumentReference | A FHIR DocumentReference resource |

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| @medplum/fhirtypes | Provides the DocumentReference type from FHIR |
| ../config | Provides the Config object for retrieving configuration settings |
