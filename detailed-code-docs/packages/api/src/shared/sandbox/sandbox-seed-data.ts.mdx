## High-level description
This code defines a sandbox environment for patient data and document references. It provides seed data for various patients, including their demographics and associated document references. The code also includes utility functions for patient matching and data retrieval.

## Code Structure
The main symbols in this code are:
- `patientMatches`: A function to check if a given patient matches the seed data.
- `seedData`: An object containing patient entries with demographics and document references.
- `getSandboxSeedData`: A function to retrieve seed data for a given patient.

These symbols are interconnected, with `patientMatches` using `getSandboxSeedData` to retrieve patient data for comparison.

## Symbols

### `patientMatches`
#### Description
This function checks if a given patient's data matches the seed data.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| patient | PatientData | The patient data to be matched |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| result | boolean | True if the patient matches seed data, false otherwise |

#### Internal Logic
1. Retrieves seed data for the patient's first name.
2. Compares various demographic fields (firstName, lastName, dob, genderAtBirth).
3. Checks if addresses match.

### `seedData`
#### Description
An object containing patient entries with demographics and document references.

### `getSandboxSeedData`
#### Description
Retrieves seed data for a given patient key.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| patientKey | string | The patient's first name (case-insensitive) |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| patientEntry | PatientEntry \| undefined | The patient's seed data if found, undefined otherwise |

## Dependencies
The code relies on several imported types and functions:
- `Address` from "@metriport/core/domain/address"
- `PatientData` from "@metriport/core/domain/patient"
- `USState` from "@metriport/shared"
- Various document reference imports for different patients

## Configuration
The code defines a `PatientEntry` type and a `seedData` object containing patient information for the sandbox environment.

## Error Handling
The code doesn't implement explicit error handling. It returns `undefined` if a patient's seed data is not found.

## API/Interface Reference
The main interface is the `seedData` object, which can be accessed using patient names as keys. The `getSandboxSeedData` function provides a way to retrieve this data programmatically.

This code serves as a crucial part of a sandbox or testing environment for a medical records system, providing realistic patient data for development and testing purposes.
