## High-level description
This code defines a Zod schema for validating streaming data from the Oura API. It ensures the data conforms to the expected structure with fields for interval, items, and timestamp.

## Symbols

### `streamingDataSchema`
#### Description
This symbol defines a Zod schema named `streamingDataSchema` used to validate the structure and types of streaming data received from the Oura API.

#### Inputs
This symbol does not take any inputs.

#### Outputs
This symbol does not return any outputs. It defines a schema used for validation.

#### Internal Logic
The schema expects an object with the following properties:
- `interval`: A number representing the time interval between data points.
- `items`: An array of objects. The schema does not enforce a specific structure for the objects within the array, allowing for flexibility in handling different types of Oura data.
- `timestamp`: A string representing the timestamp of the data.

#### Example Usage
```typescript
import { streamingDataSchema } from "./oura";

const sampleData = {
  interval: 60,
  items: [
    { heartRate: 70, timestamp: "2023-10-26T10:00:00Z" },
    { heartRate: 72, timestamp: "2023-10-26T10:01:00Z" },
  ],
  timestamp: "2023-10-26T10:02:00Z",
};

const validationResult = streamingDataSchema.safeParse(sampleData);

if (validationResult.success) {
  // Data is valid, proceed with processing
  console.log("Data is valid:", validationResult.data);
} else {
  // Data is invalid, handle the error
  console.error("Data is invalid:", validationResult.error);
}
```
