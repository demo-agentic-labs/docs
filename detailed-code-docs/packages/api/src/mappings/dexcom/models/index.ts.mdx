## High-level description
This code defines Zod schemas for common data structures used in Dexcom API responses. It includes schemas for the base response structure, base record structure, and specific record types like "evgs" (estimated glucose values) and "events".

## Code Structure
The code defines several Zod schemas. The `dexcomResp` schema represents the common structure of Dexcom API responses. The `baseRecordSchema` defines common fields for all record types. Other schemas like `recordSchema` in `evgs.ts` and `eventRecordSchema` in `events.ts` extend the `baseRecordSchema` with type-specific fields.

## References
This file is referenced by `evgs.ts` and `events.ts` to reuse and extend the base schemas.

## Symbols

### `dexcomResp`
#### Description
Defines the Zod schema for the common structure of responses from the Dexcom API.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| dexcomResp | z.ZodObject | Zod object schema representing the Dexcom API response structure |

#### Internal Logic
The schema defines three required string fields:
- `recordType`: Type of the record.
- `recordVersion`: Version of the record format.
- `userId`: Unique identifier of the user.

### `baseRecordSchema`
#### Description
Defines the Zod schema for the common fields present in all types of Dexcom records.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| baseRecordSchema | z.ZodObject | Zod object schema representing the base Dexcom record structure |

#### Internal Logic
The schema defines five required string fields:
- `systemTime`: Timestamp of the record in the system's time zone.
- `displayTime`: Timestamp of the record in the user's display time zone.
- `recordId`: Unique identifier of the record.
- `displayDevice`: Identifier of the device used for display.
- `transmitterGeneration`: Generation of the transmitter used.
