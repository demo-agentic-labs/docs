## High-level description
The code defines a Zod schema, `fitbitHeartRateResp`, to validate and type-check the heart rate data returned by the Fitbit API. It also defines a type alias, `FitbitHeartRate`, for the inferred type of the schema.

## References
- This code references the Fitbit Web API documentation for the "Get Heart Rate Time Series by Date" endpoint.
- The `FitbitHeartRate` type is used in the `src/providers/fitbit.ts` file to type-check the data fetched from the Fitbit API.

#### Internal Logic
The schema defines the expected structure of the Fitbit heart rate response, including:
- `dateTime`: A string representing the date.
- `value`: An object containing:
    - `customHeartRateZones`: An optional array of custom heart rate zones, each with:
        - `caloriesOut`: An optional number representing calories burned in the zone.
        - `max`: An optional number representing the maximum heart rate of the zone.
        - `min`: An optional number representing the minimum heart rate of the zone.
        - `minutes`: An optional number representing the minutes spent in the zone.
        - `name`: An optional string representing the name of the zone.
    - `heartRateZones`: An array of default heart rate zones, each with the same structure as `customHeartRateZones`.
    - `restingHeartRate`: An optional number representing the resting heart rate.
