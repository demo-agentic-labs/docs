## High-level description
This code defines the structure and conversion logic for patient demographics data in a medical API. It exports types and a function to transform patient data from a domain model to a Data Transfer Object (DTO) format, facilitating standardized data exchange between different parts of the application.

## Code Structure
The code defines two main types: `GenderDTO` and `DemographicsDTO`, and exports a function `dtoFromModel` to convert patient data from a domain model to the DTO format.

- `Patient` from "@metriport/core/domain/patient"
- `AddressDTO` from "./addressDTO"
- `ContactDTO` from "./contact-dto"
- `PersonalIdentifierDTO` from "./personal-identifier-dto"

## Symbols

### `GenderDTO`
#### Description
A type alias representing possible gender values.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| - | string | One of "F", "M", "O", or "U" |

### `DemographicsDTO`
#### Description
An interface defining the structure of patient demographic data.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| firstName | string | Patient's first name |
| lastName | string | Patient's last name |
| dob | string | Date of birth |
| genderAtBirth | GenderDTO | Gender at birth |
| personalIdentifiers | PersonalIdentifierDTO[] \| null \| undefined | Personal identifiers |
| address | AddressDTO[] | Patient's addresses |
| contact | ContactDTO[] \| null \| undefined | Patient's contact information |

### `dtoFromModel`
#### Description
A function that converts patient data from the domain model to the DTO format.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| patient | Pick&lt;Patient, "data"&gt; | Patient object containing demographic data |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| - | DemographicsDTO | Converted patient demographics in DTO format |

#### Internal Logic
1. Destructure the required fields from the input `patient.data` object.
2. Return a new object with the extracted fields, matching the `DemographicsDTO` structure.
