## High-level description
This code defines Data Transfer Object (DTO) types and conversion functions for CodeableConcept and Coding entities in a medical API. It provides a way to transform domain objects into DTOs, ensuring type safety and proper data structure for API responses.

## Code Structure
The code defines two main types (`CodingDTO` and `CodeableConceptDTO`) and two conversion functions (`toDTO` and `codingToDTO`). These are used to transform domain objects into DTO representations.

## References
This code references the `CodeableConcept` and `Coding` types from the `"../../../domain/medical/codeable-concept"` module.

### `toDTO`
#### Description
Converts a CodeableConcept domain object to its DTO representation.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| domain | CodeableConcept \| undefined | The domain object to convert |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| result | CodeableConceptDTO \| undefined | The converted DTO object |

#### Internal Logic
1. If the input is undefined, return undefined.
2. Create a new object with `coding` and `text` properties.
3. If `domain.coding` exists, map each coding to its DTO representation using `codingToDTO`.
4. Set the `text` property to `domain.text` or undefined if it doesn't exist.

### `codingToDTO`
#### Description
Converts a Coding domain object to its DTO representation.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| domain | Coding | The domain object to convert |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| result | CodingDTO | The converted DTO object |

#### Internal Logic
Create a new object with `system`, `code`, and `display` properties, using the nullish coalescing operator to set undefined for null values.
