## High-level description
This file defines Data Transfer Object (DTO) types and conversion functions for patient data in a medical API. It provides structures for representing patient information in a format suitable for API responses and internal data handling.

## Code Structure
The file defines several types and functions that are interconnected:
- `PatientDTO` and `InternalPatientDTO` types represent patient data for external and internal use.
- `dtoFromModel` and `internalDtoFromModel` functions convert patient domain models to their respective DTOs.

## Symbols

### `PatientDTO`
#### Description
Represents the patient data that is exposed through the API.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| facilityIds | string[] | IDs of facilities associated with the patient |
| externalId | string (optional) | External identifier for the patient |
| dateCreated | Date (optional) | Date when the patient record was created |

This type also extends `DemographicsDTO`, which likely includes personal information like name, date of birth, etc.

### `InternalPatientDTO`
#### Description
Extends `PatientDTO` with additional fields for internal use.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| externalData | PatientExternalData \| undefined | Additional external data associated with the patient |
| documentQueryProgress | DocumentQueryProgress \| undefined | Progress information for document queries |

This type also extends `BaseDTO`, which likely includes common fields like ID and eTag.

### `dtoFromModel`
#### Description
Converts a Patient domain model to a PatientDTO.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| patient | Patient | The patient domain model to convert |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| (return value) | PatientDTO | The converted PatientDTO |

#### Internal Logic
1. Extracts relevant fields from the patient's data.
2. Combines these fields with base DTO information.
3. Returns a new PatientDTO object.

### `internalDtoFromModel`
#### Description
Converts a Patient domain model to an InternalPatientDTO.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| patient | Patient | The patient domain model to convert |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| (return value) | InternalPatientDTO | The converted InternalPatientDTO |

#### Internal Logic
1. Extracts all relevant fields from the patient's data, including internal-only fields.
2. Combines these fields with base DTO information.
3. Returns a new InternalPatientDTO object.
