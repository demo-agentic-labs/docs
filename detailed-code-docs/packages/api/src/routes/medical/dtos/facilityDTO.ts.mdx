## High-level description
This code defines Data Transfer Objects (DTOs) for facilities in a medical API. It includes types for both external and internal facility representations, as well as functions to convert facility domain models to these DTOs.

## Code Structure
The code defines two main types: `FacilityDTO` and `InternalFacilityDTO`. It also includes two functions, `dtoFromModel` and `internalDtoFromModel`, which convert a `Facility` domain model to their respective DTO representations.

#### Description
Converts a `Facility` domain model to a `FacilityDTO`.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|

#### Internal Logic
1. Extracts relevant properties from the facility's data
2. Combines these properties with the base DTO (id and eTag) and the facility's OID
