## High-level description
This code defines constants for custom HTTP headers used in the Metriport API. These headers are used for version tracking, request signatures, and request identification.

## Symbols

### `VERSION_HEADER_NAME`
#### Description
A constant that defines the name of the custom HTTP header used to specify the Metriport API version.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| VERSION_HEADER_NAME | string | The string value "x-metriport-version" |

### `SIGNATURE_HEADER_NAME`
#### Description
A constant that defines the name of the custom HTTP header used for request signatures in the Metriport API.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| SIGNATURE_HEADER_NAME | string | The string value "x-metriport-signature" |

### `REQUEST_ID_HEADER_NAME`
#### Description
A constant that defines the name of the custom HTTP header used to identify individual requests in the Metriport API.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| REQUEST_ID_HEADER_NAME | string | The string value "x-metriport-request-id" |

## References
These constants are used in various parts of the Metriport API codebase:

1. In `packages/api/src/routes/medical/patient.ts`, `REQUEST_ID_HEADER_NAME` is used to set the request ID in the response header.
2. In `packages/api/src/__tests__/e2e/mapi/webhook/webhook-handler.ts`, `SIGNATURE_HEADER_NAME` is used to verify the webhook signature.
3. In `packages/api/src/app.ts`, `VERSION_HEADER_NAME` is used to set the API version in the response header.

These headers play important roles in version control, request tracking, and security (signature verification) throughout the API.
