## High-level description
The `getCxFFStatus` function retrieves the status of various HIE (Health Information Exchange) feature flags for a specific customer (CX) from AWS AppConfig. This information indicates whether features like CommonWell, CareQuality, Epic integration, or Demo Augmentation are enabled for the given customer.

## References
- `@metriport/core/external/aws/app-config`: Uses the `getFeatureFlagValueCxValues` function to fetch feature flag values from AWS AppConfig.
- `../../shared/config`: Uses the `Config` class to retrieve environment variables and configuration settings.

## Symbols

### `getCxFFStatus`
#### Description
This function retrieves the status of HIE feature flags for a specific customer from AWS AppConfig.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| cxId | string | The ID of the customer. |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| Promise\&lt;CxFeatureFlagStatus&gt; | Promise\&lt;object&gt; | A promise that resolves to an object containing the status of HIE feature flags for the customer. |

#### Internal Logic
1. Retrieves the AWS region, AppConfig application ID, AppConfig configuration ID, and environment type from the `Config` class.
2. Calls the `getFeatureFlagValueCxValues` function from `@metriport/core/external/aws/app-config` to fetch the feature flag values for the given customer ID, using the retrieved configuration parameters.
3. Returns the promise returned by `getFeatureFlagValueCxValues`.
