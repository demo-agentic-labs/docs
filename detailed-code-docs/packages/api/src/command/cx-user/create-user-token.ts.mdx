## High-level description
The `createUserToken` function generates a new user token, saves it, and returns the created token. This token is likely used for authentication or authorization purposes within the application. 

## References
- `UserToken`: This symbol is referenced for creating and representing user tokens. It is defined in the `packages/api/src/domain/user-token.ts` file.
- `saveUserToken`: This function is called to persist the generated user token. It is defined in the same file as the current code.

## Symbols

### `createUserToken`
#### Description
This asynchronous function generates a new user token, persists it using the `saveUserToken` function, and returns the created token.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| userTokenCreate | `UserTokenCreate` | An object containing the data required to create a new user token. |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| Promise\&lt;UserToken\&gt; | `Promise&lt;UserToken&gt;` | A promise that resolves to the created `UserToken` object. |

#### Internal Logic
1. **Create UserToken:** It calls `UserToken.create(userTokenCreate)` to generate a new `UserToken` instance using the provided `userTokenCreate` data.
2. **Save UserToken:** It calls `saveUserToken(userToken)` to persist the newly created `userToken`.
3. **Return UserToken:** It returns the `userToken` object.
