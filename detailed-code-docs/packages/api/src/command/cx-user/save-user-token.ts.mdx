## High-level description
The code defines a function `saveUserToken` that saves a user token to a DynamoDB table. It takes a `UserToken` object as input and returns the same object after saving it to the database.

## References
- `UserToken` from "../../domain/user-token"
- `getDB` and `docTableNames` from "../../models/db"

## Symbols
### `saveUserToken`
#### Description
This asynchronous function saves a user token to a DynamoDB table named "token".

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| userToken | `SaveUserTokenCommand` | The user token to be saved. This type is an alias for `UserToken`. |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| userToken | `UserToken` | The user token object that was saved. |

#### Internal Logic
1. It retrieves the DynamoDB document client using `getDB()`.
2. It uses the `put` method of the document client to save the `userToken` object to the table specified by `docTableNames.token`.
3. It waits for the `put` operation to complete using `.promise()`.
4. It returns the `userToken` object.

## Side Effects
- Modifies the DynamoDB table named "token" by adding or updating a user token.

## Dependencies
- `aws-sdk`: Used for interacting with DynamoDB.

## Error Handling
The code does not implement specific error handling beyond the basic exception raising mechanism provided by the AWS SDK. Any errors during the database operation will be thrown and need to be handled by the caller of this function.
