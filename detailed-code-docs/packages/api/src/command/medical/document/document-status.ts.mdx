## High-level description
The `areDocumentsProcessing` function determines if document processing operations (downloading or converting) are currently in progress for a given patient. It accepts either a `Patient` object or a `DocumentQueryProgress` object as input and returns a boolean value.

## Symbols

### `areDocumentsProcessing`
#### Description
This function checks if document processing tasks are ongoing for a patient. It handles input flexibility by accepting either a `Patient` object or a `DocumentQueryProgress` object. If a `Patient` is provided, it extracts the `documentQueryProgress` from the patient's data. It then checks if either the 'download' or 'convert' progress status is marked as 'processing'.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| param | `Patient` \| `DocumentQueryProgress` \| `undefined` |  An object representing either the patient or the document query progress. |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
|  | `boolean` | Returns `true` if document processing is in progress, otherwise `false`. |

#### Internal Logic
1. Checks if the input parameter is defined. If not, returns `false`.
2. If the input is a `Patient` object, it extracts the `documentQueryProgress` from the `data` property. Otherwise, it assumes the input is a `DocumentQueryProgress` object.
3. Utilizes the `isProcessing` function from `@metriport/core/domain/document-query` to check if either the 'download' or 'convert' progress status is 'processing'.
4. Returns `true` if either 'download' or 'convert' is in progress, otherwise returns `false`.
