## High-level description
This code defines utility functions for creating test data related to document query progress in a medical API context. It provides functions to generate mock data for `Progress` and `DocumentQueryProgress` objects, which are likely used in testing scenarios for document query operations.

## Symbols

### `makeProgress`
#### Description
This function creates a mock `Progress` object with optional seed data.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| seed | Partial&lt;DeepNullable&lt;Progress&gt;&gt; | Optional partial Progress object to seed the generated data |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| Progress | object | A Progress object with mock data |

#### Internal Logic
- Uses `faker` to generate random data for fields not provided in the seed
- Utilizes `makeOptionalNumber` function to handle optional number fields

### `makeDocumentQueryProgress`
#### Description
This function creates a mock `DocumentQueryProgress` object with optional seed data.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| seed | Partial&lt;DocumentQueryProgress&gt; | Optional partial DocumentQueryProgress object to seed the generated data |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| DocumentQueryProgress | object | A DocumentQueryProgress object with mock data |

#### Internal Logic
- Uses `makeProgress` function to generate mock data for `download` and `convert` fields
- Handles the `requestId` field, generating a UUID if not provided in the seed
