## High-level description
This file contains utility functions for creating test data related to patients in a medical system. It provides functions to generate mock patient data, personal identifiers, and patient creation commands for testing purposes.

## Symbols

### `makePersonalIdentifier()`
#### Description
Creates a mock personal identifier object with random values.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| PersonalIdentifier | object | A mock personal identifier object |

### `makePatientData(data: Partial&lt;PatientData&gt; = {})`
#### Description
Generates mock patient data, allowing partial overrides.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| data | Partial&lt;PatientData&gt; | Optional partial patient data to override defaults |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| PatientData | object | A complete mock patient data object |

### `makePatient(params: Partial&lt;Omit&lt;Patient, "data"&gt; & { data: Partial&lt;PatientData&gt; }&gt; = {})`
#### Description
Creates a mock patient object, allowing partial overrides for both patient properties and patient data.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| params | Partial&lt;Omit&lt;Patient, "data"&gt; & \{ data: Partial&lt;PatientData&gt; \}&gt; | Optional partial patient object to override defaults |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| Patient | object | A complete mock patient object |

### `makePatientCreate(params: Partial&lt;PatientCreateCmd&gt; = {})`
#### Description
Generates a mock patient creation command, allowing partial overrides.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| params | Partial&lt;PatientCreateCmd&gt; | Optional partial patient creation command to override defaults |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| PatientCreateCmd | object | A complete mock patient creation command object |

## Dependencies
The file imports from several modules:
- `@faker-js/faker` for generating random data
- `@metriport/shared` for US state data
- `@metriport/core/domain/patient` for patient-related types
- `dayjs` for date manipulation
- Local modules for base domain and address creation

## Notes
- The functions use the `faker` library to generate random data for various fields.
- The `makePatientData` function includes several optional fields that can be populated with more specific data if needed.
- The `makePatient` and `makePatientCreate` functions build upon `makePatientData` to create more complete objects for testing purposes.
