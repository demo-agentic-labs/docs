## High-level description
This file contains constant data structures used for testing demographics-related functionality in the Metriport API. It defines sample patient demographic data, including core demographics, link demographics, and consolidated link demographics, as well as a complete patient object.

## Symbols

### `coreDemographics`
#### Description
A constant object of type `LinkDemographics` that represents a set of core demographic information for a patient.

#### Inputs
None (constant)

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| coreDemographics | LinkDemographics | An object containing various demographic details |

#### Internal Logic
- Defines personal information such as date of birth, gender, names, addresses, phone numbers, emails, driver's licenses, and SSNs.
- Uses `JSON.stringify()` with sorted keys for consistent string representation of complex objects (names, addresses, driver's licenses).

### `linkDemographics`
#### Description
Another constant object of type `LinkDemographics`, representing an alternative set of demographic information.

#### Inputs
None (constant)

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| linkDemographics | LinkDemographics | An object containing various demographic details |

#### Internal Logic
Similar to `coreDemographics`, but with different values and fewer entries in some fields.

### `consolidatedLinkDemographics`
#### Description
A constant object representing consolidated link demographics, omitting some fields present in `LinkDemographics`.

#### Inputs
None (constant)

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| consolidatedLinkDemographics | Partial&lt;LinkDemographics&gt; | An object containing consolidated demographic details |

#### Internal Logic
Contains a subset of the fields present in `linkDemographics`, specifically excluding `dob` and `gender`.

### `patientDemo`
#### Description
A constant object of type `PatientDemoData` that represents a more structured set of patient demographic data.

#### Inputs
None (constant)

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| patientDemo | PatientDemoData | An object containing structured patient demographic data |

#### Internal Logic
Defines patient information in a format that matches the `PatientDemoData` type, including personal details, addresses, personal identifiers, and contact information.

### `patient`
#### Description
A constant object of type `Patient` that represents a complete patient record, including demographic data and metadata.

#### Inputs
None (constant)

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| patient | Patient | An object representing a complete patient record |

#### Internal Logic
Combines `patientDemo` data with `consolidatedLinkDemographics` and adds additional fields required for a complete `Patient` object, such as `id`, `cxId`, `createdAt`, `updatedAt`, `facilityIds`, and `eTag`.

## Notes
- This file is likely used for testing purposes, providing consistent and varied demographic data for different test scenarios.
- The use of `JSON.stringify()` with sorted keys ensures consistent string representation of complex objects, which can be useful for comparison in tests.
- The file demonstrates the structure and relationships between different demographic data types used in the Metriport API.
