## High-level description
This file is a TypeScript configuration file (tsconfig.json) for the ihe-gateway-sdk package. It defines compiler options and project settings for TypeScript, extending from a recommended configuration and adding strict type-checking rules to ensure high-quality, type-safe code.

## Symbols

### CompilerOptions
#### Description
The `compilerOptions` object specifies various TypeScript compiler settings for the project.

#### Internal Logic
1. Extends the recommended TypeScript configuration.
2. Sets the target ECMAScript version to ES2020.
3. Enables source map generation for debugging.
4. Generates declaration files and their corresponding maps.
5. Enables decorator metadata and experimental decorators.
6. Incorporates strict type-checking rules from @tsconfig/strictest.
7. Specifies the output directory for compiled files.

### Include and Exclude
#### Description
These arrays define which files should be included in and excluded from the compilation process.

#### Internal Logic
- Includes all files in the `./src` directory.
- Excludes the `dist` and `node_modules` directories.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| @tsconfig/recommended | Provides a base recommended TypeScript configuration |

## Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| target | string | "es2020" | Specifies the ECMAScript target version |
| sourceMap | boolean | true | Generates corresponding .map files |
| declaration | boolean | true | Generates corresponding .d.ts files |
| declarationMap | boolean | true | Generates a sourcemap for each corresponding '.d.ts' file |
| emitDecoratorMetadata | boolean | true | Enables experimental support for emitting type metadata for decorators |
| experimentalDecorators | boolean | true | Enables experimental support for decorators |
| allowUnusedLabels | boolean | false | Disallows unused labels |
| allowUnreachableCode | boolean | false | Disallows unreachable code |
| noFallthroughCasesInSwitch | boolean | true | Reports errors for fallthrough cases in switch statements |
| noImplicitOverride | boolean | true | Ensures overriding members in derived classes are marked with an override modifier |
| noUnusedLocals | boolean | true | Reports errors on unused local variables |
| checkJs | boolean | true | Reports errors in .js files |
| exactOptionalPropertyTypes | boolean | true | Enables strict checking of property initialization |
| noImplicitReturns | boolean | true | Reports error when not all code paths in function return a value |
| noPropertyAccessFromIndexSignature | boolean | true | Enforces using indexed accessors for keys declared using an indexed type |
| noUncheckedIndexedAccess | boolean | true | Adds undefined to a type when accessed using an index |
| outDir | string | "./dist" | Specifies the output directory for compiled files |

Note: This configuration extends the recommended TypeScript configuration and incorporates additional strict type-checking rules from @tsconfig/strictest to ensure high-quality, type-safe code.
