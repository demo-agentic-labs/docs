## High-level description
This file defines the schemas and types for patient discovery requests in an IHE (Integrating the Healthcare Enterprise) gateway SDK. It uses Zod for schema validation and defines both inbound and outbound patient discovery request structures.

## Code Structure
The code defines two main schemas: `outboundPatientDiscoveryReqSchema` for outbound requests and `inboundPatientDiscoveryReqSchema` for inbound requests. These schemas extend a base schema (`patientDiscoveryDefaultSchema`) and define the structure for patient discovery requests. The file also exports corresponding TypeScript types derived from these schemas.

## Symbols

### `patientDiscoveryDefaultSchema`
#### Description
A base schema for patient discovery requests, extending the `baseRequestSchema` and including a `patientResource` field.

### `outboundPatientDiscoveryReqSchema`
#### Description
Defines the schema for outbound patient discovery requests, extending `patientDiscoveryDefaultSchema` with additional fields.

#### Internal Logic
Adds fields for `gateways`, `principalCareProviderIds`, `patientId`, and `cxId` to the base schema.

### `OutboundPatientDiscoveryReq`
#### Description
TypeScript type inferred from `outboundPatientDiscoveryReqSchema`.

### `inboundPatientDiscoveryReqSchema`
#### Description
Defines the schema for inbound patient discovery requests, based on `patientDiscoveryDefaultSchema` but omitting the `cxId` field.

### `InboundPatientDiscoveryReq`
#### Description
TypeScript type inferred from `inboundPatientDiscoveryReqSchema`.
