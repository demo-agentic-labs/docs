## High-level description
This file, `env.ts`, provides utility functions for accessing environment variables in a Lambda function context. It includes functions to retrieve environment variables, check the current environment type (production, sandbox, or staging), and contains deprecated functions that are now replaced by core functionality.

## Symbols

### `getEnv`
#### Description
A deprecated function that retrieves the value of an environment variable.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| name | string | The name of the environment variable to retrieve |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| value | string \| undefined | The value of the environment variable, or undefined if not set |

### `getEnvOrFail`

## Notes
- The `getEnv` and `getEnvOrFail` functions are marked as deprecated, suggesting that users should use the core functionality instead.
- The environment type checking functions (`isProduction`, `isSandbox`, `isStaging`) rely on the `getEnvType` function from the core utility.
