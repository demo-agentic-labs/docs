## High-level description
This Java code demonstrates how to use the Metriport API to retrieve patient information. It includes two methods: one for getting all patients and another for retrieving a specific patient's details using the Metriport SDK.

## Symbols

### `getAllPatients`
#### Description
This method retrieves a list of all patients using the Metriport API.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| args | String[] | Command-line arguments (unused in this method) |

#### Internal Logic
1. Loads environment variables using Dotenv.
2. Builds a Metriport client with API key and base URL from environment variables.
3. Calls the Metriport API to list all patients.
4. Prints the response to the console.

### `getSpecificPatient`

This method retrieves information for a specific patient using the Metriport API.

#### Internal Logic

## Notes
1. The code assumes that the API_KEY and BASE_URL are set in the environment or a .env file.
2. Error handling is not implemented in this code. In a production environment, proper error handling should be added.
3. The `getSpecificPatient` method doesn't specify a patient ID, which might lead to unexpected behavior or errors. It should be modified to accept a patient ID as a parameter.
4. Both methods are static and don't return any value. They simply print the API response to the console.
5. The `args` parameter is not used in either method, suggesting that these methods might be intended to be called from a main method or another part of the application.
