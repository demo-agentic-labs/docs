## High-level description
This Java code contains unit tests for the Metriport SDK, specifically testing the functionality to retrieve patient information. It includes two test methods: one for retrieving all patients and another for retrieving a specific patient by ID.

## Symbols

### `TestGetPatient`
#### Description
This is a test class that contains methods to test the patient retrieval functionality of the Metriport SDK.

### `getAllPatients()`
#### Description
This test method demonstrates how to retrieve a list of all patients using the Metriport SDK.

#### Internal Logic
1. Loads environment variables using Dotenv.
2. Builds a Metriport client instance with API key and base URL from environment variables.
3. Calls the `list()` method on the patient resource to retrieve all patients.
4. Prints the response to the console.

### `getSpecificPatient()`
#### Description
This test method demonstrates how to retrieve a specific patient by ID using the Metriport SDK.

#### Internal Logic
1. Loads environment variables using Dotenv.
2. Builds a Metriport client instance with API key and base URL from environment variables.
3. Calls the `get()` method on the patient resource, passing a patient ID from environment variables.
4. Prints the response to the console.

## Configuration
The code relies on environment variables for configuration:
| Option | Type | Description |
|:-------|:-----|:------------|
| API_KEY | String | API key for authenticating with the Metriport API |
| BASE_URL | String | Base URL for the Metriport API |
| PATIENT_ID | String | ID of a specific patient to retrieve in the `getSpecificPatient()` test |

Note: These environment variables are expected to be defined in a .env file or set in the system environment.
