## High-level description
This code defines types and utility functions for handling patient data in a Master Patient Index (MPI) system. It provides methods to convert between different patient data representations and includes types for patient data structures.

## Symbols

### `PatientMPI`
#### Description
A type that represents a patient in the Master Patient Index system. It combines the `id` from the `Patient` type with all the fields from `PatientData`.

### `patientToPatientMPI`
#### Description
A function that converts a `Patient` object to a `PatientMPI` object.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| patient | Patient | The patient object to convert |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| PatientMPI | PatientMPI | The converted PatientMPI object |

#### Internal Logic
1. Extracts the `id` from the input `patient` object.
2. Spreads the `data` property of the input `patient` object.
3. Returns a new object combining the `id` and the spread `data`.

### `patientMPIToPartialPatient`
#### Description
A function that converts a `PatientMPI` object to a partial `Patient` object, including only the `id` and `data` fields.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| patient | PatientMPI | The PatientMPI object to convert |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| PartialPatient | Pick&lt;Patient, "id" \| "data"&gt; | A partial Patient object with id and data fields |

#### Internal Logic
1. Destructures the `id` from the input `patient` object.
2. Collects all other properties into the `data` object using the rest operator.
3. Returns a new object with `id` and `data` properties.
