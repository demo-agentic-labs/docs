## High-level description
This code defines a `Config` class that provides a centralized way to access configuration settings and environment variables for the application. It includes methods to check the environment type, retrieve various configuration values, and access sensitive information like API keys and certificates.

## Code Structure
The `Config` class contains static methods that retrieve environment variables using the `getEnvVar` and `getEnvVarOrFail` functions imported from the `env-var` module. These methods provide access to different configuration settings, such as environment types, URLs, AWS regions, and various API keys and certificates.

## Symbols

#### Description
A static class that provides methods to access configuration settings and environment variables.

#### Internal Logic
- Defines constants for different environment types (PROD_ENV, DEV_ENV, SANDBOX_ENV, STAGING_ENV)

- Retrieves various configuration values from environment variables

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| getEnvVar | Function to retrieve environment variables |
| getEnvVarOrFail | Function to retrieve environment variables, throwing an error if not found |

## Error Handling
The `getEnvVarOrFail` function is used to retrieve critical environment variables, throwing an error if they are not defined. This ensures that the application fails fast if required configuration is missing.
