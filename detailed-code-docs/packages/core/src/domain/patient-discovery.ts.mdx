## High-level description
This code defines types and interfaces related to patient discovery in a healthcare system. It includes structures for scheduling patient discovery, defining discovery parameters, and managing the discovery process across different healthcare networks like CommonWell and Carequality.

## Code Structure
The main symbols defined are `ScheduledPatientDiscovery` and `DiscoveryParams`. These are used to structure data for patient discovery operations. The code also includes TODO comments indicating planned changes to remove certain fields.

## Symbols

### `ScheduledPatientDiscovery`
#### Description
This type represents the structure for scheduling a patient discovery operation.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| requestId | string | Unique identifier for the discovery request |
| facilityId | string | Identifier for the healthcare facility |
| orgIdExcludeList | string[] | Optional list of organization IDs to exclude |
| rerunPdOnNewDemographics | boolean | Optional flag to rerun discovery on new demographics |
| forceCommonwell | boolean | Optional flag to force CommonWell discovery (to be removed) |
| forceCarequality | boolean | Optional flag to force Carequality discovery (to be removed) |

### `DiscoveryParams`
#### Description
This type defines the parameters for a patient discovery operation.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| requestId | string | Unique identifier for the discovery request |
| facilityId | string | Identifier for the healthcare facility |
| startedAt | Date | Timestamp when the discovery started |
| rerunPdOnNewDemographics | boolean | Flag to rerun discovery on new demographics |
