## High-level description
This file contains example data for various vital sign observations in FHIR format. It defines several partial Observation objects representing different vital signs such as heart rate, temperature, respiratory rate, weight, blood pressure, and height.

## Symbols

### `vitalSignObservationTemplate`
#### Description
A template for a vital sign observation, specifically for pain severity.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| N/A | Partial&lt;Observation&gt; | This is a constant, not a function |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| vitalSignObservationTemplate | Partial&lt;Observation&gt; | A partial Observation object for pain severity |

#### Description

#### Inputs/Outputs

## Notes
1. All examples follow a similar structure, including:
   - A `code` object with `text` and `coding` properties
   - An `effectiveDateTime` field
   - A `valueQuantity` object with `value`, `unit`, and `system` properties

2. The `vitalSignObservationTemplate` is more detailed than the others, including `resourceType`, `status`, and `category` fields.

3. All examples use standard coding systems:
   - LOINC (http://loinc.org) for observation codes
   - UCUM (http://unitsofmeasure.org) for units of measure

4. The observations cover a range of vital signs commonly measured in clinical settings.

5. These examples are likely used for testing or as templates in the FHIR to CDA conversion process, as indicated by the file location in the `__tests__` directory.
