## High-level description
This code defines functions and constants for creating and manipulating FHIR FamilyMemberHistory resources. It provides a factory function to create a basic FamilyMemberHistory object and includes predefined constants for specific family member types.

## Symbols

### `makeFamilyMemberHistory`
#### Description
A factory function that creates a FamilyMemberHistory object with default values and allows for partial overrides.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| params | Partial&lt;FamilyMemberHistory&gt; | Optional. Partial FamilyMemberHistory object to override default values. |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| FamilyMemberHistory | FamilyMemberHistory | A complete FamilyMemberHistory object. |

#### Internal Logic
1. Spreads the result of `makeBaseDomain()` (not defined in this file)
2. Sets default values for `resourceType`, `status`, `relationship`, and `sex`
3. Overrides any values provided in the `params` argument

### `motherFamilyMemberHistory`
#### Description
A constant representing a partial FamilyMemberHistory object for a mother.

### `naturalBrotherCode`
#### Description
A constant representing the coding for a natural brother relationship.

## Configuration
The code defines several configuration constants for FHIR coding systems:
| System | Description |
|:-------|:------------|
| http://terminology.hl7.org/CodeSystem/v3-RoleCode | Used for family relationship codes |
| http://hl7.org/fhir/administrative-gender | Used for sex/gender codes |
| http://snomed.info/sct | Used for SNOMED CT codes (in the mother example) |

These systems are used to provide standardized codes for relationships and sex in the FamilyMemberHistory objects.
