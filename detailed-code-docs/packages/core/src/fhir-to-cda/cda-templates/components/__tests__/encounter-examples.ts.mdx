## High-level description
This file contains example data for encounters, practitioners, and locations used in testing FHIR to CDA conversion. It defines partial objects of FHIR resources (Encounter, Practitioner, and Location) with sample data that can be used to create more complete test scenarios.

## Symbols

### `encounter1`
#### Description
A partial Encounter object representing a finished urgent care office visit.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| encounter1 | Partial&lt;Encounter&gt; | A partial FHIR Encounter resource |

#### Internal Logic
- Defines status as "finished"
- Sets class and type with CPT code "99214"
- Includes period start and end times
- Specifies reason codes for the encounter

### `practitioner1`
#### Description
A partial Practitioner object representing a family medicine physician.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| practitioner1 | Partial&lt;Practitioner&gt; | A partial FHIR Practitioner resource |

#### Internal Logic
- Includes identifier (NPI)
- Sets name, address, and telecom information
- Specifies qualification as a family medicine physician

### `practitioner2`
#### Description
Another partial Practitioner object, similar to practitioner1 but with a different name.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| practitioner2 | Partial&lt;Practitioner&gt; | A partial FHIR Practitioner resource |

#### Internal Logic
- Inherits properties from practitioner1
- Overrides name and qualification

### `location1`
#### Description
A partial Location object representing an urgent care medical office.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| location1 | Partial&lt;Location&gt; | A partial FHIR Location resource |

#### Internal Logic
- Sets name and address
- Specifies type as "Urgent Care"
