## High-level description
This file contains utility functions and example data for creating and manipulating FHIR resources, specifically Practitioners, Locations, and Encounters. It is primarily used for testing purposes, providing a way to generate mock data with consistent structures and example values.

## Code Structure
The file defines several functions to create FHIR resources and multiple constant objects with example data. These functions and constants are used across different test files to create consistent test scenarios.

## Symbols

### `makePractitioner`
#### Description
Creates a Practitioner resource with the given parameters.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| params | Partial&lt;Practitioner&gt; | Partial Practitioner object to merge with the base Practitioner |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| Practitioner | Practitioner | A FHIR Practitioner resource |

### `makeLocation`
#### Description
Creates a Location resource with the given parameters.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| params | Partial&lt;Location&gt; | Partial Location object to merge with the base Location |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| Location | Location | A FHIR Location resource |

### `makeEncounter`
#### Description
Creates an Encounter resource with the given parameters and optional IDs.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| params | Partial&lt;Encounter&gt; | Partial Encounter object to merge with the base Encounter |
| ids | \{ enc?: string; pract?: string; loc?: string \} | Optional IDs for the Encounter, Practitioner, and Location |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| Encounter | Encounter | A FHIR Encounter resource |

### Constants
The file defines several constants with example data:
- `practitionerNameZoidberg`: An example HumanName object
- `exampleAddress`: An example Address object
- `exampleType` and `exampleType2`: Example Encounter type codings
- `exampleHospitalization` and `exampleHospitalization2`: Example hospitalization data
- `exampleReasonCode` and `exampleReasonCode2`: Example reason codes
- `exampleDiagnosis` and `exampleDiagnosis2`: Example diagnosis references
