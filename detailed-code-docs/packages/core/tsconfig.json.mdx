## High-level description
This file is a TypeScript configuration file (tsconfig.json) for the core package of a project. It defines compiler options and specifies which files should be included or excluded from the TypeScript compilation process. The configuration extends a recommended base configuration and includes additional strict type-checking options.

## Configuration
The tsconfig.json file contains various configuration options for the TypeScript compiler. Here's a breakdown of the key settings:

| Option | Type | Value | Description |
|:-------|:-----|:------|:------------|
| extends | string | "@tsconfig/recommended/tsconfig.json" | Extends the recommended TypeScript configuration |
| target | string | "es2020" | Specifies the ECMAScript target version |
| sourceMap | boolean | true | Generates corresponding .map files |
| declaration | boolean | true | Generates corresponding .d.ts files |
| declarationMap | boolean | true | Generates sourcemaps for .d.ts files |
| emitDecoratorMetadata | boolean | true | Enables experimental support for emitting type metadata for decorators |
| experimentalDecorators | boolean | true | Enables experimental support for decorators |
| allowUnusedLabels | boolean | false | Disallows unused labels |
| allowUnreachableCode | boolean | false | Disallows unreachable code |
| noFallthroughCasesInSwitch | boolean | true | Reports errors for fallthrough cases in switch statements |
| noImplicitOverride | boolean | true | Ensures overriding members are marked with an override modifier |
| noUnusedLocals | boolean | true | Reports errors on unused local variables |
| checkJs | boolean | true | Enables type checking on JavaScript files |
| exactOptionalPropertyTypes | boolean | true | Enables strict checking of optional property assignments |
| noImplicitReturns | boolean | true | Ensures all code paths in a function return a value |
| noUncheckedIndexedAccess | boolean | true | Adds undefined to any un-declared field in the type of an index signature |
| outDir | string | "./dist" | Specifies the output directory for compiled files |
| resolveJsonModule | boolean | true | Allows importing .json files |

### Include and Exclude
The configuration also specifies which files to include and exclude from the compilation process:

- Include: "./src" and "./src/**/*.json"
- Exclude: "dist" and "node_modules"

## Dependencies
This configuration extends the recommended TypeScript configuration:
| Dependency | Purpose |
|:-----------|:--------|
| @tsconfig/recommended | Provides a base recommended TypeScript configuration |

## Notes
1. The configuration includes several strict type-checking options, some of which are from the @tsconfig/strictest configuration.
2. The `declarationMap` option is enabled to allow importers to jump to the source code.
3. The configuration is set up to compile TypeScript files in the `src` directory and its subdirectories, including JSON files.
4. The compiled output will be placed in the `dist` directory.
5. This configuration is tailored for a project that requires strict type checking and targets modern JavaScript environments (ES2020).
