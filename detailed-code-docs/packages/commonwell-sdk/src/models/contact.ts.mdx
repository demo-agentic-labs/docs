## High-level description
This code defines the structure and validation schema for contact information in the CommonWell SDK. It uses the Zod library for schema validation and includes enums for contact use codes and system codes, as well as a schema for contact details.

## Code Structure
The code defines two main enums (`ContactUseCodes` and `ContactSystemCodes`) and their corresponding Zod schemas. It then defines a `contactSchema` that represents a contact's details. The `Contact` type is inferred from this schema.

## Symbols

### `ContactUseCodes`
#### Description
An enum that defines the possible use cases for a contact point, based on the HL7 FHIR R4 value set.

#### Internal Logic
Includes standard FHIR codes and additional codes (`unspecified`) that may need review before use.

### `contactUseCodesSchema`
#### Description
A Zod schema that validates the `ContactUseCodes` enum values.

### `ContactSystemCodes`
#### Description
An enum that defines the possible system types for a contact point, based on the HL7 FHIR R4 value set.

### `contactSystemCodesSchema`
#### Description
A Zod schema that validates the `ContactSystemCodes` enum values.

### `contactSchema`
#### Description
A Zod schema that defines the structure and validation rules for a contact object.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| use | ContactUseCodes (optional) | The use case for the contact point |
| system | string (optional) | The system type for the contact point |
| value | string (optional) | The actual contact information (e.g., phone number, email) |
| period | periodSchema (optional) | The time period for which this contact is/was valid |

### `Contact`
#### Description
A TypeScript type inferred from the `contactSchema`, representing a contact object.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| zod | Used for schema definition and validation |
| ./period | Imports the `periodSchema` for the contact's valid period |
