## High-level description
This code defines a Zod schema for a patient organization in the CommonWell SDK. It creates a structure for validating and typing patient organization data, which includes optional fields for type, reference, and display.

## Symbols

### `patientOrganizationSchema`
#### Description
This constant defines a Zod schema for a patient organization. It specifies the structure and validation rules for patient organization data.

#### Inputs
The schema doesn't have direct inputs, but it defines the shape of the data it validates.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| type | string \| null \| undefined | The type of the patient organization |
| reference | string \| null \| undefined | A reference identifier for the patient organization |
| display | string \| null \| undefined | A display name or label for the patient organization |

#### Internal Logic
The schema uses Zod's `z.object()` to create an object schema with three optional and nullable string fields: `type`, `reference`, and `display`. Each field is defined using `z.string().optional().nullable()`, which means they can be strings, undefined, or null.
