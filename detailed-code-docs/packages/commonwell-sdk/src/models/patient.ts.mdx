## High-level description
This code defines the structure and types for a Patient resource in the CommonWell Health Alliance API. It uses Zod for schema validation and includes definitions for patient data, links, and related responses.

## Code Structure
The main symbol is the `patientSchema`, which defines the structure of a Patient resource. Other schemas like `patientLinksSchema`, `patientSearchRespSchema`, `patientNetworkLinkRespSchema`, and `patientLinkRespSchema` are used to define related structures and responses.

## Symbols

### `patientLinksSchema`
#### Description
Defines the structure for links associated with a patient.

#### Internal Logic
It includes various optional link types such as self, networkLink, person, personMatch, upgrade, and downgrade.

### `patientSchema`
#### Description
Defines the structure of a Patient resource.

#### Internal Logic
It includes fields for active status, identifiers, provider, demographics details, links, and facilities.

### `Patient`
#### Description
A type alias for the inferred type from `patientSchema`.

### `patientSearchRespSchema`
#### Description
Defines the structure for a patient search response.

#### Internal Logic
It includes a message, embedded patient array, and links.

### `PatientSearchResp`
#### Description
A type alias for the inferred type from `patientSearchRespSchema`.

### `patientNetworkLinkRespSchema`
#### Description
Defines the structure for a patient network link response.

#### Internal Logic
It includes embedded network links and self link.

### `PatientNetworkLinkResp`
#### Description
A type alias for the inferred type from `patientNetworkLinkRespSchema`.

### `patientLinkRespSchema`
#### Description
Defines the structure for a patient link response.

#### Internal Logic
It includes embedded patient links and self link.

### `PatientLinkResp`
#### Description
A type alias for the inferred type from `patientLinkRespSchema`.
