## High-level description
This file defines the structure and types for identifiers used in the CommonWell Health Alliance API. It includes schemas for identifier use codes, general identifiers, and strong identifiers, which are used to uniquely identify patients and other entities in healthcare systems.

## Symbols

### `identifierUseCodesSchema`
#### Description
Defines an enumeration of valid identifier use codes as specified by CommonWell Alliance.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| identifierUseCodesSchema | z.ZodEnum | Zod schema for identifier use codes |

### `IdentifierUseCodes`
#### Description
Type definition derived from the `identifierUseCodesSchema`.

### `identifierSchema`
#### Description
Defines the structure of an identifier used in the CommonWell system.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| identifierSchema | z.ZodObject | Zod schema for identifiers |

### `Identifier`
#### Description
Type definition derived from the `identifierSchema`.

### `strongIdSchema`
#### Description
Defines a more strict version of the identifier schema where `system` and `key` are required.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| strongIdSchema | z.ZodObject | Zod schema for strong identifiers |

### `StrongId`
#### Description
Type definition derived from the `strongIdSchema`.

## Notes
- The file uses Zod for schema definition, which provides both runtime validation and static type inference.
- The schemas are based on the CommonWell Alliance specification for identifiers.
- Strong identifiers are a subset of regular identifiers with stricter requirements.
