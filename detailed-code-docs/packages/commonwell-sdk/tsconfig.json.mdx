## High-level description
This file is a TypeScript configuration file (tsconfig.json) for the CommonWell SDK package. It defines compiler options and project settings to ensure strict type-checking, enable specific features, and configure the compilation process for the TypeScript code in the project.

## Code Structure
The configuration is structured as a JSON object with three main sections: "extends", "compilerOptions", and file inclusion/exclusion settings.

## Symbols

### Root Configuration Object
#### Description
The root object contains the entire TypeScript configuration for the CommonWell SDK package.

#### Internal Logic
1. Extends the recommended TypeScript configuration.
2. Defines specific compiler options.
3. Specifies which files to include and exclude in the compilation process.

### extends
#### Description
Extends the recommended TypeScript configuration, providing a base set of compiler options.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| @tsconfig/recommended/tsconfig.json | string | Path to the recommended TypeScript configuration file |

### compilerOptions
#### Description
Defines specific compiler options for the TypeScript compiler, including target ECMAScript version, source map generation, decorator support, and strict type-checking rules.

#### Internal Logic
1. Sets the target ECMAScript version to ES2020.
2. Enables source map and declaration file generation.
3. Configures decorator metadata and experimental decorators.
4. Implements strict type-checking rules, many of which are from @tsconfig/strictest.
5. Specifies the output directory for compiled files.

### include
#### Description
Specifies which files should be included in the compilation process.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| ./src | string | Path to the source directory |

### exclude
#### Description
Specifies which files or directories should be excluded from the compilation process.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| dist | string | Output directory |
| node_modules | string | Node.js modules directory |

## Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| target | string | "es2020" | Specifies the ECMAScript target version |
| sourceMap | boolean | true | Generates corresponding .map files |
| declaration | boolean | true | Generates corresponding .d.ts files |
| declarationMap | boolean | true | Generates a sourcemap for each corresponding .d.ts file |
| emitDecoratorMetadata | boolean | true | Emits design-type metadata for decorated declarations in source |
| experimentalDecorators | boolean | true | Enables experimental support for decorators |
| allowUnusedLabels | boolean | false | Disallows unused labels |
| allowUnreachableCode | boolean | false | Disallows unreachable code |
| noFallthroughCasesInSwitch | boolean | true | Reports errors for fallthrough cases in switch statements |
| noImplicitOverride | boolean | true | Ensures overriding members are marked with an override modifier |
| noUnusedLocals | boolean | true | Reports errors on unused local variables |
| checkJs | boolean | true | Reports errors in .js files |
| exactOptionalPropertyTypes | boolean | true | Enables strict checking of property initialization |
| noImplicitReturns | boolean | true | Reports error when not all code paths in function return a value |
| noPropertyAccessFromIndexSignature | boolean | true | Enforces using indexed accessors for keys declared using an indexed type |
| noUncheckedIndexedAccess | boolean | true | Adds undefined to a type when accessed using an index |
| outDir | string | "./dist" | Specifies the output directory for compiled files |

This configuration sets up a strict and modern TypeScript environment for the CommonWell SDK package, ensuring type safety and enabling various TypeScript features to improve code quality and developer experience.
