## High-level description
This GitHub Actions workflow, named "Fern Check", is designed to run the `fern check` command on the repository. It triggers on pull requests, merge group events, and pushes to the master and develop branches, ensuring that Fern API specifications are validated at key points in the development process.

## Symbols

### Workflow: "Fern Check"
#### Description
This workflow defines a single job that installs the Fern CLI tool and runs the `fern check` command to validate Fern API specifications.

#### Internal Logic
1. The workflow is triggered on pull requests, merge group events, and pushes to the master and develop branches.
2. It runs on the latest Ubuntu runner.
3. The job consists of three steps:
   - Checkout the repository
   - Install the Fern CLI tool globally using npm
   - Run the `fern check` command

This workflow provides a simple but crucial check for maintaining the integrity of Fern API specifications in the project. It ensures that any changes to the API specifications are validated before merging into the main branches or during pull requests, helping to catch potential issues early in the development process.
