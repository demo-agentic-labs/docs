## High-level description
This YAML file (`fern/generators.yml`) defines configuration for generating SDKs for different programming languages (Python, Java, and Node.js) using the Fern API framework. It specifies the generator versions, output locations, and various configuration options for each SDK.

## Code Structure
The file is structured into groups, each representing a different SDK or testing configuration. The main groups are `python-sdk`, `java-sdk`, `node-sdk`, and `test`. Each group contains one or more generator configurations.

## Symbols

### default-group
#### Description
Specifies the default group to use when generating SDKs. In this case, it's set to "test".

### groups
#### Description
Defines the different SDK and testing configurations.

#### Internal Logic
Each group contains a `generators` key with an array of generator configurations. These configurations specify the generator to use, its version, output location, and various options.

### python-sdk
#### Description
Configures the generation of the Python SDK.

#### Internal Logic
- Uses the `fernapi/fern-python-sdk` generator version 0.11.2
- Outputs to PyPI with the package name "metriport"
- Configures the client class name as "Metriport"
- Sets Pydantic configuration and enables improved imports
- Specifies the GitHub repository for the SDK

### java-sdk
#### Description
Configures the generation of the Java SDK.

#### Internal Logic
- Uses the `fernapi/fern-java-sdk` generator version 0.5.20
- Outputs to Maven with the coordinate "com.metriport:metriport-java"
- Configures the client class name as "Metriport"
- Specifies the GitHub repository for the SDK

### node-sdk
#### Description
Configures the generation of the Node.js SDK.

#### Internal Logic
- Uses the `fernapi/fern-typescript-node-sdk` generator version 0.9.0
- Outputs to npm with the package name "@metriport/api-sdk"
- Configures the namespace export as "Metriport" and disables the serde layer
- Specifies the GitHub repository for the SDK

### test
#### Description
Configures local generation of SDKs for testing purposes.

#### Internal Logic
- Includes configurations for Python, Java, and TypeScript SDKs
- Outputs to local file system paths for each SDK
- Uses specific versions of generators for each language
- Sets various configuration options similar to the main SDK configurations

## Dependencies
The file relies on the Fern API framework and its generators for different programming languages.

| Dependency | Purpose |
|:-----------|:--------|
| fernapi/fern-python-sdk | Generates Python SDK |
| fernapi/fern-java-sdk | Generates Java SDK |
| fernapi/fern-typescript-node-sdk | Generates Node.js SDK |

## Configuration
The file itself serves as a configuration for the Fern API generators. Each generator configuration includes various options:

| Option | Type | Description |
|:-------|:-----|:------------|
| name | string | The name of the generator to use |
| version | string | The version of the generator |
| output | object | Specifies the output location and related settings |
| config | object | Contains generator-specific configuration options |
| github | object | Specifies the GitHub repository for the SDK |
