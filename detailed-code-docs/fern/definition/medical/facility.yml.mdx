## High-level description
This YAML file defines the API specification for managing medical facilities in Metriport. It outlines various endpoints for creating, retrieving, updating, listing, and deleting facilities, along with the data structures used for requests and responses.

## Code Structure
The file is structured into two main sections: `service` and `types`. The `service` section defines the API endpoints, while the `types` section describes the data structures used in the API.

## Symbols

### service
#### Description
Defines the base path, authentication requirement, and endpoints for the Facility API.

#### Internal Logic
- Sets the base path to `/medical/v1/facility`
- Requires authentication for all endpoints
- Defines five endpoints: create, get, update, list, and delete

### create endpoint
#### Description
Creates a new Facility in Metriport.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| body | BaseFacility | The facility details to be created |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| response | Facility | The created facility details |

### get endpoint
#### Description
Retrieves a specific Facility by its ID.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| id | string | The ID of the facility to retrieve |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| response | Facility | The retrieved facility details |

### update endpoint
#### Description
Updates an existing Facility.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| id | string | The ID of the facility to update |
| body | BaseFacility | The updated facility details |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| response | Facility | The updated facility details |

### list endpoint
#### Description
Lists all Facilities associated with the organization.

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| response | ListFacilitiesResponse | Array of all facilities |

### delete endpoint
#### Description
Deletes a specific Facility by its ID.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| id | string | The ID of the facility to delete |

### BaseFacility type
#### Description
Defines the basic properties of a Facility.

#### Properties
| Name | Type | Description |
|:-----|:-----|:------------|
| name | string | The name of the Facility |
| npi | string | The 10-digit National Provider Identifier |
| tin | optional&lt;string&gt; | The Taxpayer Identification Number |
| active | optional&lt;boolean&gt; | Whether the Facility is currently active |
| address | commons.Address | The address of the Facility |

### Facility type
#### Description
Extends BaseFacility with an additional `id` property.

#### Properties
| Name | Type | Description |
|:-----|:-----|:------------|
| id | string | The unique identifier for the Facility |

### ListFacilitiesResponse type
#### Description
Defines the response structure for the list endpoint.

#### Properties
| Name | Type | Description |
|:-----|:-----|:------------|
| facilities | list&lt;Facility&gt; | List of Facility objects |

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| commons | Imports common types, specifically the Address type |

## API/Interface Reference
| Endpoint | Method | Request | Response | Description |
|:---------|:-------|:--------|:---------|:------------|
| /medical/v1/facility | POST | BaseFacility | Facility | Creates a new Facility |
| /medical/v1/facility/{id} | GET | - | Facility | Retrieves a specific Facility |
| /medical/v1/facility/{id} | PUT | BaseFacility | Facility | Updates an existing Facility |
| /medical/v1/facility | GET | - | ListFacilitiesResponse | Lists all Facilities |
| /medical/v1/facility/{id} | DELETE | - | - | Deletes a specific Facility |

This API specification provides a comprehensive set of operations for managing medical facilities in Metriport, including creation, retrieval, updating, listing, and deletion of facilities. It uses consistent data structures and provides examples for each endpoint to facilitate understanding and implementation.
